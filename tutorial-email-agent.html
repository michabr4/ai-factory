Your First Agent — Step by Step

Build an Email
Summary Agent

Read your Outlook inbox and get a plain-English summary of what matters — built entirely through vibe coding in about 30 minutes.

⏱ ~30 minutes
📝 No coding experience needed

What You'll Build

A Python script that:

📧
Connects to Outlook

🤖
Summarizes with AI

✅
Shows You What Matters
Outputs a prioritized daily brief with action items

Before You Start

1

Windsurf (or Cursor) installed

Your AI-powered code editor. Download at windsurf.com if you haven't already.

2

Python installed

Most Macs have it. Open Terminal and type python3 --version. If you see a version number, you're good. If not, ask the AI to help you install it.

3

A Microsoft 365 work or personal account

The one you use for Outlook email. You'll log in through a browser window during setup — no passwords stored in code.

4

An OpenAI API key (or alternative)

Sign up at platform.openai.com. The first few dollars of usage are often free. Your key stays in a private .env file — never in code.

1

Create Your Project

Open Windsurf and create a new folder for your agent. Then open the chat panel and give it your first prompt:

Paste this into the AI chat

"Create a new Python project called email-summary-agent. Set up a virtual environment, a requirements.txt with the packages I'll need for reading Outlook email via Microsoft Graph and summarizing with OpenAI, and a .env file for my secrets. Also create a .gitignore that keeps the .env file and virtual environment out of version control."

What the AI will do: Create a project folder, virtual environment, requirements.txt (with msal, requests, openai, python-dotenv), a blank .env template, and a .gitignore.

Security note: Your .env file will hold your API keys. It should NEVER be committed to Git or shared. The .gitignore prevents this automatically.

2

Register an Azure App

To read your Outlook email, you need to tell Microsoft "this app is allowed to access my mail." This is a one-time setup.

Paste this into the AI chat

"Walk me through registering an Azure AD app for Microsoft Graph with delegated permissions to read my email. I need Mail.Read permission. I want to use device code flow so I don't need a redirect URL. Give me step-by-step instructions with screenshots descriptions."

The AI will guide you through these steps. Here's the summary:

a

Go to Azure Portal → App Registrations and click "New registration"

b

Name it "Email Summary Agent", select "Accounts in this organizational directory only" (or personal if using personal Outlook), skip the redirect URI

c

After creation, copy the Application (client) ID and Directory (tenant) ID from the overview page

d

Go to API Permissions → Add a permission → Microsoft Graph → Delegated → Mail.Read

e

Go to Authentication → Advanced settings → Allow public client flows → Yes (this enables device code flow)

What are these IDs? Think of them as your app's name tag. The Client ID says "I'm the Email Summary Agent" and the Tenant ID says "I belong to this organization." They're not secret, but your API keys are — those go in .env only.

Now paste the IDs into your .env file:

# .env — NEVER commit this file
AZURE_CLIENT_ID=paste-your-client-id-here
AZURE_TENANT_ID=paste-your-tenant-id-here
OPENAI_API_KEY=paste-your-openai-key-here
3

Build the Email Reader

Now you'll ask the AI to write the code that connects to Outlook and pulls your emails.

Paste this into the AI chat

"Create a Python file called email_reader.py that: 1) Loads my Azure credentials from .env, 2) Uses MSAL with device code flow to authenticate me — I'll approve via browser, 3) Calls Microsoft Graph API to get my 20 most recent emails from today, 4) Returns a clean list with sender name, subject, date, and a preview of the body text (first 500 characters, stripped of HTML). Make sure no credentials are hardcoded."

What happens when you run it: A URL and a code appear in your terminal. Open the URL in your browser, paste the code, and sign in with your Microsoft account. After that, the script can read your email. This is called "device code flow" — it's the same way you sign into a streaming app on a TV.

Then test it

"Run email_reader.py and show me the output. If there are any errors, fix them."

Expect iteration here. Authentication can be finicky — you might hit a permissions error or a tenant issue. Just paste the error message back to the AI and say "fix this." This is normal vibe coding workflow.

4

Add the AI Summarizer

Now you'll connect an LLM to turn raw email data into a useful daily brief.

Paste this into the AI chat

"Create a Python file called summarizer.py that: 1) Takes the list of emails from email_reader.py, 2) Sends them to the OpenAI API (use gpt-4o-mini to keep costs low) with a prompt that asks for: a) A 3-sentence overview of my inbox today, b) A prioritized list of emails that need my attention with why, c) Any action items or deadlines mentioned, d) Emails I can probably ignore. 3) Load the OpenAI key from .env, never hardcode it. 4) Print the summary in a clean, readable format to the terminal."

Here's roughly what the AI's system prompt to the LLM will look like (the AI will write this for you):

# The AI will craft something like this:

system_prompt = """
You are an executive email assistant. Analyze the following
emails and provide:

1. INBOX OVERVIEW — 3 sentences summarizing today's email
2. NEEDS ATTENTION — Emails requiring action, ranked by
   urgency, with a one-line reason for each
3. ACTION ITEMS — Specific tasks or deadlines mentioned
4. LOW PRIORITY — Emails that are informational only

Be concise. Use the sender's name, not their email address.
Flag anything with a deadline in the next 48 hours.
"""
Then wire it together

"Now create a main.py that imports email_reader and summarizer, fetches my emails, summarizes them, and prints the result. Add a nice header with today's date and the number of emails processed."

"Run main.py for me"

6

Make It Better

Once the basic version works, here are enhancements you can ask for — one at a time:

Cache Login

So you don't re-authenticate every time

"Add MSAL token caching so I only need to sign in once, and it refreshes automatically."

Filter Timeframe

Summarize last hour, today, or this week

"Add a command-line flag so I can run: python main.py --hours 4 to only summarize the last 4 hours of email."

Save to File

Keep a daily log you can reference

"Save each summary to a markdown file in a /summaries folder, named by date like 2025-05-21.md."

Schedule It

Run automatically every morning

"Help me set up a cron job on my Mac to run this every weekday at 7:30 AM and save the summary."

Common Issues & Fixes

"AADSTS700016: Application not found"

Your Client ID in .env doesn't match your Azure app. Double-check you copied it correctly from the Azure Portal overview page.

"Insufficient privileges" or "403 Forbidden"

You need to add the Mail.Read permission in Azure Portal → API Permissions. If your org requires admin consent, ask your IT admin to approve it.

"RateLimitError" from OpenAI

You've hit your usage limit. Either wait a minute and retry, or add billing at platform.openai.com/account/billing.

Emails return but summary is weird

The HTML in your emails may be cluttering the LLM input. Tell the AI: "Strip all HTML tags from the email body before sending to OpenAI. Only send plain text."

Universal fix: When something breaks, copy the full error message, paste it into the AI chat, and say "I got this error, fix it." That's the vibe coding way.
